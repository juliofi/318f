1. Migração do backend de Spring Boot para Firebase
Substituímos a dependência do backend anterior pela SDK modular do Firebase, centralizando a comunicação diretamente pelo app web.
Configuramos o arquivo src/services/firebase.ts com as credenciais providas pelo console do Firebase, inicializando o app com os serviços necessários.
Exponibilizamos somente o módulo de autenticação (getAuth) e removemos integrações desnecessárias, garantindo carregamento seguro pelo front.
Benefícios:
Elimina a dependência do servidor Spring Boot para autenticação básica.
Reduz latência e simplifica o stack operacional, já que o cliente fala direto com o Firebase.
Facilita o uso de outros produtos Firebase no futuro, pois a base já está configurada.

2. Implementação do login com autenticação Firebase
Atualizamos src/pages/AuthPage para consumir signInWithEmailAndPassword e signInWithPopup, tratando estados de loading e erros.
Incluímos mensagens de falha e desabilitamos botões quando o Firebase não está configurado, evitando ações inválidas.
Adicionamos suporte ao GoogleAuthProvider para login social, mantendo o redirecionamento para /dashboard após sucesso.
Benefícios:
Usuários conseguem autenticar com credenciais reais ou Google sem depender de mocks.
Feedback imediato melhora a UX e ajuda no debug de configurações.
Fluxo pronto para incorporar outras checagens (guards, contextos) mantendo a base de auth unificada.
